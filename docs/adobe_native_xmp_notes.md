# Adobe Native Mask XMP Notes

## Current Decision

Dust Mask Repair does not currently write Adobe Camera Raw / Lightroom native
local-mask XMP.

The implemented XMP sidecar is host-neutral and uses the custom
`https://dust-mask-repair.local/ns/1.0/` namespace. It is safe to read back with
`read_mask_xmp()` and safe to use as this project's interchange format.

## Why Native Adobe Masks Are Not Enabled Yet

Adobe's public Camera Raw namespace documents standard raw/edit settings under
`http://ns.adobe.com/camera-raw-settings/1.0/`, but that namespace alone is not
enough to describe an arbitrary pixel mask.

Adobe's current Photoshop API migration guide describes advanced Camera Raw XMP
mask workflows as XMP plus separate mask payloads and digests. It notes that
mask files are typically binary files generated by Adobe Camera Raw. That means
writing plausible-looking `crs:*` XMP without the corresponding verified mask
payload would be unsafe.

References:

- Adobe Camera Raw namespace: https://developer.adobe.com/xmp/docs/xmp-namespaces/crs/
- Adobe Photoshop API edit operations, XMP with masks: https://developer.adobe.com/firefly-services/docs/photoshop/guides/photoshop-v2-beta/v1-to-v2/edit-operations

## Required Before Enabling

To implement a native Adobe adapter, use a separate validation slice with:

- a real Lightroom / Camera Raw XMP sidecar containing a local mask;
- any associated Adobe-generated binary mask payload;
- a test image and expected import behavior;
- a round-trip check showing that Adobe accepts the generated output.

Until then, `require_adobe_native_mask_support()` fails closed with
`NotImplementedError` instead of producing a file that only appears compatible.
